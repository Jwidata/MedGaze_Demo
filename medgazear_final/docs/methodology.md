# Methodology

## Overview

MedGazeAR Final implements a non-clinical research pipeline for modeling ROI attention behavior during radiology-style image review. The system links CT segmentation context with synthetic Tobii-like gaze data, engineered gaze features, explainable attention rules, behavior-learning models, and leakage-aware evaluation.

The methodology is designed for thesis research and future real eye-tracking validation. It is not designed for clinical diagnosis or deployment.

## Research Question

The recommended research question is:

> Can segmentation-derived ROI context and gaze-derived scanpath features be used to build an auditable framework for modeling radiology review attention behavior, and what validation is required before applying the framework to real eye-tracking data?

## System Pipeline

The implemented pipeline has ten main stages:

1. DICOM metadata audit.
2. CT/SEG matching.
3. ROI geometry extraction.
4. Synthetic gaze generation.
5. Gaze degradation simulation.
6. ROI, scanpath, temporal, quality, and geometry feature extraction.
7. Explainable rule-based ROI attention classification.
8. Rule-recovery and leakage auditing.
9. Behavior-learning model training and robustness auditing.
10. Cognitive-load proxy analysis.

## Stage 1: DICOM Metadata Audit

The DICOM scanner reads metadata without relying on clinical inference. It inventories CT series and segmentation objects so that later stages can reason about available imaging context.

Current generated audit outputs report:

- 250,273 readable DICOM files.
- 243,414 CT slices.
- 1,017 CT series.
- 6,859 SEG objects.
- 0 invalid files in the scanned run.

Primary outputs:

- `outputs/dicom_audit/dicom_inventory.csv`
- `outputs/dicom_audit/ct_series_summary.csv`
- `outputs/dicom_audit/seg_inventory.csv`
- `outputs/dicom_audit/dicom_audit_summary.md`

## Stage 2: CT/SEG Matching

Segmentation objects are matched to referenced CT series. The purpose is to ensure that generated ROI geometry can be tied to image context.

Current generated matching outputs report:

- 6,859 SEG objects.
- 6,844 strict matches.
- 10 partial matches.
- 5 unmatched missing CT cases.

Primary outputs:

- `outputs/roi_geometry/ct_seg_match_table.csv`
- `outputs/roi_geometry/ct_seg_match_summary.md`

## Stage 3: ROI Geometry Extraction

The segmentation-derived ROI stage converts SEG context into geometry features that can be used by synthetic gaze generation and feature extraction. ROI-level context includes bounding boxes, centroid position, ROI area, normalized position, slice index, number of ROIs on a slice, and ROI density context.

The full ROI geometry pool includes:

- 39,890 ROI/frame rows.
- 6,844 SEG objects.
- 871 LIDC-derived patients.

This gives the synthetic gaze stage a realistic spatial substrate even though the gaze itself is simulated.

## Stage 4: Synthetic Gaze Generation

The synthetic gaze generator creates behavior-labeled gaze sessions over sampled ROI geometry. Each session selects a reader profile, case, ROI, hidden behavior label, duration, and trajectory pattern.

Current generated synthetic run:

- 2,000 sessions.
- 1,034,799 raw gaze samples.
- 1,923 unique sampled ROI IDs.
- 871 available patients represented in the source ROI geometry.

Hidden behavior labels include:

- `expert_like_systematic_review`
- `focused_roi_confirmation`
- `partial_near_miss_review`
- `missed_roi_search`
- `skipped_slice`
- `high_load_fragmented_review`

Reader profiles include expert-like, fast-confirming, partial, noisy, high-load, and distracted behavior templates.

Important limitation: these labels are generated by the simulation policy. They are not measured clinical labels.

## Stage 5: Tobii-Like Gaze Degradation

The synthetic gaze generator applies Tobii-like degradation to make the raw samples less idealized. The degradation model includes:

- Screen-space jitter.
- Calibration offset.
- Time-dependent drift.
- Dropout samples.
- Blink samples.
- Invalid bursts.
- Outside-CT samples.
- UI-glance samples.

The current synthetic quality report includes:

- Mean validity ratio: approximately `0.959`.
- Dropout ratio: approximately `0.025`.
- Blink ratio: approximately `0.009`.
- Invalid burst ratio: approximately `0.007`.
- Outside-CT ratio: approximately `0.046`.
- UI-glance ratio: approximately `0.049`.

These parameters support simulation realism but do not replace measured Tobii validation.

## Stage 6: Feature Extraction

Raw gaze samples are aggregated into ROI/session-level features. The feature groups are:

ROI coverage features:

- Dwell time inside ROI.
- Dwell time near ROI.
- Gaze hit count inside and near ROI.
- Fixation count inside and near ROI.
- Time to first ROI fixation.
- Valid gaze time on the ROI slice.

Scanpath/search features:

- Scanpath length.
- Scanpath length on ROI slice.
- Gaze dispersion.
- Gaze entropy.
- Number of gaze clusters.
- Background gaze ratio.
- ROI revisit count.
- Near-ROI revisit count.
- Slice transition count.
- Adjacent slice toggle count.
- Search-to-confirmation ratio.
- Late ROI discovery flag.

Temporal features:

- Mean and maximum fixation duration.
- Fixation duration variance.
- Saccade-like ratio.
- Fixation-like ratio.
- First-half and second-half ROI attention ratio.
- Delayed attention score.

Quality features:

- Gaze validity ratio.
- Dropout ratio.
- Blink ratio.
- Invalid burst ratio.
- Outside-CT ratio.
- Jitter.

Geometry/context features:

- ROI area.
- Bounding-box width and height.
- ROI center.
- Normalized ROI position.
- Number of ROIs on slice.
- ROI density context.

Primary outputs:

- `outputs/features/behavior_feature_table.csv`
- `outputs/features/feature_schema.md`
- `outputs/features/feature_quality_report.md`

## Stage 7: Rule-Based Attention Classification

The rule attention engine uses interpretable thresholds to classify whether an ROI was reviewed, weakly reviewed, not reviewed, or not evaluated.

The rule engine considers:

- Valid gaze time on the ROI slice.
- Gaze validity ratio.
- Inside-ROI dwell time.
- Inside-ROI hit count.
- Inside-ROI fixation count.
- Near-ROI dwell time.
- Near-ROI hit count.

Current distribution:

- `reviewed`: 17
- `weakly_reviewed`: 1,347
- `not_reviewed`: 500
- `not_evaluated`: 136

The rule system is explainable and useful as a reference baseline. It is not clinical truth.

## Stage 8: Rule-Recovery And Leakage Auditing

Rule-recovery audits test whether ML models are merely reproducing handcrafted rule logic. This is important because gaze features can contain direct or indirect proxies for the rule thresholds.

Current rule-recovery report includes:

- Full feature set macro F1: `0.976`.
- No direct rule features macro F1: `0.970`.
- Geometry-only negative control macro F1: `0.251`.
- Ultra-deleaked geometry/context-only macro F1: `0.251`.
- Shuffled-label control macro F1: `0.236`.

Interpretation: geometry-only and shuffled-label controls are low, which is good. However, high performance after removing direct rule features suggests strong indirect temporal and scanpath proxies. This should be reported as simulation behavior, not independent clinical discovery.

## Stage 9: Behavior Learning

Behavior-learning models classify hidden synthetic behavior labels from engineered features. The training pipeline excludes forbidden identifiers and labels, including session ID, reader ID, case ID, ROI ID, hidden label, rule attention status, and related rule evidence fields.

Models evaluated include:

- `DummyClassifier`
- `ExtraTreesClassifier`
- `LogisticRegression`
- `MLPClassifier`
- `RandomForestClassifier`
- `XGBoostClassifier`

The model-selection process uses:

- Training split for fitting candidate models.
- Validation macro F1 for model selection.
- Validation balanced accuracy as tie-breaker.
- Untouched held-out test split for final reporting.

Current behavior-learning result:

- Selected model: `XGBoostClassifier`.
- Validation macro F1: `0.819`.
- Validation balanced accuracy: `0.822`.
- Held-out test macro F1: `0.812`.
- Held-out test balanced accuracy: `0.808`.

This result supports the internal learnability of the generated behavior classes. It does not validate real-world radiologist behavior prediction.

## Stage 10: Robustness Auditing

The behavior robustness audit tests whether the behavior classifier remains stable under grouped splits, hard feature ablation, noise stress, and new-seed proxy testing.

Current audit results report very high performance under several conditions, including case group split, reader-held-out split, hard feature ablation, noise stress, and new-seed proxy testing.

Interpretation: the synthetic behavior classes are highly separable. This is useful for testing the pipeline, but it also reinforces the need for real Tobii validation before external claims.

## Stage 11: Cognitive-Load Proxy Analysis

The cognitive-load proxy combines percentile-ranked gaze behavior and signal-quality features into low, medium, and high proxy labels.

Features include:

- Gaze dispersion.
- Scanpath length.
- ROI revisits.
- Slice transitions.
- Delayed attention.
- Fixation variance.
- Saccade-like behavior.
- Dropout, blink, and invalid-burst ratios.
- Outside-CT and background-gaze ratios.

The output labels are:

- `low_load_proxy`
- `medium_load_proxy`
- `high_load_proxy`

These are not validated cognitive-load labels. Validation would require additional ground truth such as pupil diameter, NASA-TLX, task difficulty ratings, or expert workload annotations.

## Reproducibility

The pipeline is script-driven and produces explicit CSV, JSON, Markdown, and model artifacts under `outputs/`. Tests are available under `tests/` and cover the major feature extraction, synthetic generation, DICOM scanning, rule auditing, behavior learning, and cognitive proxy components.

## Methodological Position

The methodology is strongest when presented as a synthetic-to-real research framework. The current prototype demonstrates how to build, audit, and interpret a gaze-aware ROI attention modeling system before collecting real eye-tracking data.
